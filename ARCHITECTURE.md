# 아키텍처 — 볼트 설계 설명

## 왜 하나의 볼트인가

이 프로젝트는 **팀 공유 볼트 하나**만 운영한다.

| 공간 | 위치 |
|------|------|
| 팀 공유 | `karts-mothership/` (이 저장소) |
| 개인 지식 자산 | 각자 로컬에서 독립 관리 |

개인 레퍼런스, 학습 자료, 스크랩은 볼트에 섞지 않는다.  
볼트에는 **팀이 함께 만드는 것**만 담는다.

## 폴더별 역할

```
00. Inbox/
```
분류 전 임시 메모. 아이디어, 에러 기록, 소스 후보 등 일단 여기에 던진다.  
정리되면 적절한 폴더로 이동.

```
01. Projects/ai-animation-2026/
```
프로젝트의 심장. 구조는 다음과 같다:

| 폴더 | 내용 | 편집 권한 |
|------|------|------|
| `_Project Overview.md` | 목표·일정·소스 현황 | 팀 공통 |
| `Synopsis/` | 공통 소스 애니메이션 설명 (인물·씬·세계관) | 팀 공통 |
| `Setup/` | Blender 소스 파일 경로 + ComfyUI 파이프라인 세팅 | 팀 공통 |
| `Moods/member-X/` | 멤버별 스타일 방향 + 개인 레퍼런스 노트 | 본인만 |

```
02. Daily/
```
작업일마다 짧은 일지. 파일명 규칙: `YYYY-MM-DD-이름.md`  
진행 상황, 막힌 부분, 발견한 것을 기록한다.

```
03. Permanent Notes/
```
프로젝트가 끝난 후에도 남길 것들.  
파이프라인 노하우, 재사용 가능한 프롬프트, 다음 프로젝트에 쓸 인사이트.

```
90. Settings/Templates/
```
노트 템플릿 모음. 새 노트를 만들 때 여기서 불러다 쓴다.

## 그래프 뷰 활용 전략

**레퍼런스는 파일 하나에 몰아쓰지 않는다.**  
`Moods/member-X/` 안에서 레퍼런스 하나 = 노트 하나로 쪼갠다.

```
Moods/member-A/
├── _Style Direction.md    ← 개별 노트들을 wikilink로 연결
├── ref-블레이드러너.md    ← [[cyberpunk]] [[noir]] 등 키워드 연결
└── ref-아키라.md          ← [[cyberpunk]] [[animation]] 등 키워드 연결
```

공통 키워드 노트(예: `[[cyberpunk]]`)가 여러 멤버의 레퍼런스 노트에 연결되면  
그래프 뷰에서 **예상 외의 교차점**이 드러난다.

## 영상 파일 규칙

ComfyUI 렌더 결과물(mp4 등)은 볼트 밖에 둔다.  
볼트에는 **파일 경로와 메모만** 기록한다.

```
# 볼트 안 (OK)
렌더 결과 경로: /Users/xxx/renders/animation-2026/member-A-take1.mp4

# 볼트 안 (금지)
member-A-take1.mp4 파일 자체
```
